# Codex App

> **Source / 原文来源**: https://developers.openai.com/codex/app/  
> **Compiled for study on / 整理日期**: 2026-04-07

The Codex app is a focused desktop experience for working on Codex threads in parallel, with built-in worktree support, automations, and Git functionality.

Codex app 是一款专注于桌面端的应用，可用于并行处理 Codex 线程，并内置 worktree 支持、自动化和 Git 功能。

ChatGPT Plus, Pro, Business, Edu, and Enterprise plans include Codex. Learn more about [what’s included](/codex/pricing).

ChatGPT Plus、Pro、Business、Edu 和 Enterprise 计划均包含 Codex。了解更多[包含内容](/codex/pricing)。

The Codex app is available on macOS (Apple Silicon).

Codex app 可在 macOS（Apple Silicon）上使用。

1. Download and install the Codex app

   下载并安装 Codex app

   Download the Codex app for Windows or macOS.

   下载适用于 Windows 或 macOS 的 Codex app。

2. Open Codex and sign in

   打开 Codex 并登录

   Once you downloaded and installed the Codex app, open it and sign in with your ChatGPT account or an OpenAI API key.

   下载并安装 Codex app 后，打开它并使用你的 ChatGPT 账户或 OpenAI API 密钥登录。

   If you sign in with an OpenAI API key, some functionality such as [cloud threads](about:/codex/prompting#threads) might not be available.

   如果使用 OpenAI API 密钥登录，某些功能（如[云线程](about:/codex/prompting#threads)）可能不可用。

3. Select a project

   选择项目

   Choose a project folder that you want Codex to work in.

   选择你希望 Codex 处理的文件夹。

If you used the Codex app, CLI, or IDE Extension before you’ll see past projects that you worked on.

如果你之前使用过 Codex app、CLI 或 IDE 扩展，你会看到过去处理过的项目。

4. Send your first message

   发送你的第一条消息

   After choosing the project, make sure **Local** is selected to have Codex work on your machine and send your first message to Codex.

   选择项目后，确保已选择 **Local**，以便 Codex 在你的机器上工作，然后向 Codex 发送第一条消息。

   You can ask Codex anything about the project or your computer in general. Here are some examples:

   你可以向 Codex 询问关于项目或你电脑的任何事情。以下是一些示例：

   If you need more inspiration, check out the [explore section](/codex/explore).

   如果你需要更多灵感，请查看[探索页面](/codex/explore)。

   If you’re new to Codex, read the [best practices guide](/codex/learn/best-practices).

   如果你是 Codex 新手，请阅读[最佳实践指南](/codex/learn/best-practices)。

---

### Multitask across projects / 跨项目多任务处理

跨项目多任务处理

Run multiple tasks in parallel and switch quickly between them.

并行运行多个任务并在它们之间快速切换。

### Built-in Git tools / 内置 Git 工具

内置 Git 工具

Review diffs, comment inline, stage or revert chunks, and commit without leaving the app.

查看差异、行内评论、暂存或回退代码块，无需离开应用即可完成提交。

### Worktrees for parallel tasks / 用于并行任务的 Worktrees

用于并行任务的 Worktrees

Isolate changes of multiple Codex threads using built-in Git worktree support.

利用内置的 Git worktree 支持，隔离多个 Codex 线程的更改。

### Skills support / Skills 支持

Skills 支持

Give your Codex agent additional capabilities and reuse skills across App, CLI, and IDE Extension.

为你的 Codex 代理赋予额外能力，并在 App、CLI 和 IDE 扩展之间复用 skills。

### Automations / 自动化

自动化

Pair skills with automations to automate recurring tasks in the background. Codex adds findings to the inbox, or automatically archives runs if there’s nothing to report.

将 skills 与自动化配对，在后台自动执行重复性任务。Codex 会将结果添加到收件箱，如果没有需要报告的内容，则会自动归档运行记录。

### Built-in terminal / 内置终端

内置终端

Open a terminal per thread to test your changes, run dev servers, scripts, and custom commands.

为每个线程打开一个终端，以测试更改、运行开发服务器、脚本和自定义命令。

### Local environments / 本地环境

本地环境

Define worktree setup scripts and common project actions for easy access.

定义 worktree 设置脚本和常用项目操作，以便快速访问。

### Sync with the IDE extension / 与 IDE 扩展同步

与 IDE 扩展同步

Share Auto Context and active threads across app and IDE sessions.

在应用和 IDE 会话之间共享 Auto Context 和活动线程。

### MCP support / MCP 支持

MCP 支持

Connect your Codex agent to additional services using MCP.

使用 MCP 将你的 Codex 代理连接到其他服务。
