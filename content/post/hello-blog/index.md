## 为什么开始写博客

作为一个后端开发，我大部分时间都在和代码、日志、监控系统打交道。但越深入学习，越发现一个现象：**很多知识只有在尝试向别人解释时，才真正属于自己**。

写博客的目的很简单：

1. **强制输出倒逼输入** — 只有写得出来，才算是真的理解了
2. **建立个人技术资产** — 面试、交流、复盘时，有一个可检索的知识库
3. **找到同路人** — 技术领域的很多问题，往往一篇文章就能帮到人

## 这个博客会写什么

我目前的主业是 **Go 后端开发**，同时也在往 **AI 应用** 方向转型。所以博客的内容也会围绕这两个交集展开：

### Go 后端工程
- 高并发场景下的工程实践
- 微服务架构设计与演进
- 性能分析和调优经验

### AI 应用开发
- 用大模型 API 构建实际应用（不只是调用接口）
- AI Agent 的设计与工具链（如 MCP）
- 从后端视角看 AI 产品的工程化挑战

### 工程化思考
- 代码之外的技术决策
- 从 0 到 1 做产品的经验教训
- 工具、流程和团队协作的最佳实践

## 为什么选择 Hugo

在决定写博客之前，我对比了多个平台：

| 方案 | 优点 | 缺点 |
|------|------|------|
| 公众号/知乎 | 流量大 | 平台封闭，内容不完全属于自己 |
| Notion/Obsidian | 写作体验好 | 作为公开博客展示力不够 |
| Hugo + Vercel | 完全可控、免费、加载快 | 需要一点技术门槛 |

因为我本身就是技术出身，**自己部署和维护一个博客不仅没有门槛，反而是一个加分项**。最终选定了 **Hugo** 作为静态站点生成器，配合 **Stack 主题** 和 **Vercel** 部署。

这个组合的优势：

- **Markdown 原生支持**，代码高亮极其出色
- **构建速度极快**（Hugo 本身就是 Go 写的）
- **完全免费托管**，且自动 CI/CD
- **自定义程度高**，不会被平台的算法和样式限制
